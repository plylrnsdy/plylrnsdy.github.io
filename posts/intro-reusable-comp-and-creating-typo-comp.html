    # 【译】介绍可重用组件并展示如何创建字体排版组件

    原文作者: Thu Nghiem

    原文地址: https://dev.to/nghiemthu/an-introduction-about-reusable-components-and-creating-typography-component-4c6oopen in new window

    React 变得如此流行的原因之一是它可重用的组件。过去几年里，设计系统这一概念在 Web 开发者中也逐渐变得流行。

    我发现很多人在学习 React 的过程中犯了一个错误：他们直接去学习 Redux 并开始构建复杂应用。他们忘记了去学习基础。

    由于这个原因，他们不知道为什么他们要使用 React。是因为它很有用或者仅仅是它很流行？

    在这篇文章里，我们将会看到什么是可重用组件，以及为什么它是开始学习 React 的最好方法之一。我们还将一起构建一个字体排版组件。

    # 什么是可重用组件？

    当一个组件被使用超过一次时，它就是可重用的。例如，在一个列表中，我们不想让一个列表项被使用超过一次。所以我们必须使该组件可重用。

    但是，可重用组件不仅仅是列表中的一个列表项。可重用组件的另外一些例子是按钮和输入框。它们是全局组件，因为它们可以在任何地方被使用。

    有些是可重用的，但不一定要能在任何地方被使用。一个例子是，<tr>、<th>、<td> 可以在 <table> 中重复使用，但不能（也不应该）在其他地方使用。

    你可能已经使用了可重用组件。例如，如果你使用 BEM 命名，你可以看到 Block 的名字是全局组件，而 Element 的名字是局部组件。

    当涉及到 React 时，可重用组件变得更加令人兴奋。

    # 为什么你应该关心他们

    在这一点上，你可能已经看到了可重用组件的好处。但这里有一个列表：

    # 效率

    你不再需要把时间花在思考像素和重复做同样的事情上。你可以依靠你的可重用组件来节省时间。这意味着你有更多的时间来提高质量，更快地完成你的应用程序，并减少成本。

    # 一致

    在你的应用程序中拥有一致性比你想象的更重要。随着你的用户开始使用你的应用程序，他们将开始了解它。他们将开始找到一种模式。

    一个具有一致性的应用程序将帮助你的用户更快地找到信息，并减少困惑。

    # 可维护

    假设你的设计人员决定更改按钮中的内边距。现在，你必须搜索每个有 <button> 的位置，打开每个 CSS 文件，并尝试找到内边距的位置。

    这样工作量很大。因此，如果你有可重用组件，你只需要在一个地方更改它，而不用到处手动修改。

    # 避免重复代码

    重复的代码不是一件坏事，因为它使你的应用程序更加灵活。但是，如果你必须重复写三次以上的代码就不是一件好事了。使用可重用组件可以帮助你避免每次都复制你的代码。

    # 如何制作一个好的可重用组件

    构建一个可重用组件可能是很棘手的。这里有几件你要注意的事情：

    # 组件应该是通用的

    举个例子，Button 组件不应该知道当前的主题。相反，应用程序应该告诉 Button 组件它是哪个主题。

    不正确的例子

    const Button = ({ children, ...props }) => {
  const { state } = useContext(AppContext);
  return (
    <button
      className={cn({
        "button--theme-dark": state.theme === "dark",
      })}
      {...props}
    >
      {children}
    </button>
  );
};

    在这个例子中，我们在 Button 组件中从 AppContext 获取全局状态。这意味着我们在 Button 组件和应用程序之间建立了一种依赖关系。因此，该组件只能在应用程序上下文中重用，而我们要避免这种情况。

    正确的例子

    const Button = ({  theme, children, ...props }) => {
  return (
    <button
      className={cn({
        "button--theme-dark": theme === 'dark',
      })}
      {...props}
    >
      {children}
    </button>
  );
};

    这个例子中的按钮是独立的，可以在任何应用程序中使用。这就是我们想要的。

    # 可扩展的

    组件应该足够灵活，以便你可以在以后可以轻松地添加更多配置。

    在这个例子中，不应该有布尔值属性 hasPrimaryColor，而应该是属性：color="primary"。一些其他的属性也不应该是 boolean，比如：size、varient……

    不正确的例子

    const Button = ({ hasPrimaryColor, children, ...props }) => {
  return (
    <button
      className={cn({
        "button--color-primary": hasPrimaryColor,
      })}
      {...props}
    >
      {children}
    </button>
  );
};

    正确的例子

    const Button = ({ color, children, ...props }) => {
  return (
    <button
      className={cn({
        "button--color-primary": color === "primary",
      })}
      {...props}
    >
      {children}
    </button>
  );
};

    # 简单

    组件越是复杂，就越难维护。你可能会听过这些术语：无状态组件和有状态组件。大多数情况下，无状态组件比有状态组件更简单。

    但差异在哪里呢？嗯……这值得单独写一篇文章。但基本上，如果你能把逻辑移到组件之外，并保持它的通用性，那么你就应该这样做 🙂

    # 第 2 步：导入字体

    你可以去谷歌字体open in new window选择你想要的。在我们的案例中，我们使用 Ubuntuopen in new window。

    你可以通过在 <head> 内使用 <link> 标签导入。

    # 第 3 步：使用字体并重置默认样式

    让我们来重置一些默认的样式，并使用我们的字体。通过重置默认值，我们可以自由地赋予它我们自己的样式，而不用关心默认值是什么。

    在我们的例子中，让我们移除默认的内边距和外边距。在其他的组件中，可能还需要重置 border、background-color、text-decoration……

  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
}

    # 第 4 步：创建一个字体排版组件

    一定要记得给你的组件传递 ...props，这样我们就不会丢失默认属性open in new window。

    import React from "react";
import cn from "classnames";
import "./typography.css";

// 定义组件支持的 HTML 标签
const variantsMapping = {
  h1: "h1",
  h2: "h2",
  h3: "h3",
  h4: "h4",
  h5: "h5",
  h6: "h6",
  subheading1: "h6",
  subheading2: "h6",
  body1: "p",
  body2: "p",
};

// 创建一个函数式组件，接收参数：
// variant: 选择的 html 标签
// color: 选择的颜色
// children: 传入组件内部的节点
// ...props: 组建的默认属性
const Typography = ({ variant, color, children, ...props }) => {
// 如果变体存在于 variantsMapping，那么我们就使用它。
// 否则，用 p 标签代替
  const Component = variant ? variantsMapping[variant] : "p";

  return (
    <Component
      className={cn({
        [`typography--variant-${variant}`]: variant,
        [`typography--color-${color}`]: color,
      })}
      {...props}
    >
      {children}
    </Component>
  );
};

export default Typography;

    在制作了字体排版组件后，我们可以看到，制作可重用的组件是很困难的，但通常可以为你在未来节省很多时间。而且也是开始学习 React 或 Vue 的一个好方法。
